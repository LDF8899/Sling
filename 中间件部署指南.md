# 中间件部署指南

## 概述

本机有两个 MySQL 实例：

| 实例 | 版本 | 端口 | 用途 |
|------|------|------|------|
| Windows MySQL | 8.0.33 | 3306 | 本机旧服务（建议停用） |
| Docker MySQL | 8.0.46 | 3307 | 项目数据库 |

**当前选择：停掉 Windows MySQL 权限不够，所以把 Docker MySQL 改到 3307，避开冲突。**

## 给其他项目改端口（提示词）

如果你之前有其他本地项目用的 Docker MySQL（端口 3306），用这个提示词让 Claude 帮你改：

> 项目数据库之前连的 Docker MySQL on port 3306，现在 Docker MySQL 移到 3307 了（Windows 自带 MySQL 占着 3306）。帮我找到项目里所有 MySQL 连接配置，把 port 从 3306 改成 3307，其他不变。

## 一键启动所有中间件

以**管理员身份**运行终端：

```bash
# 先停掉 Windows MySQL（避免和下面的 compose 冲突）
net stop MySQL
sc config MySQL start=demand

# 有 docker-compose 的话，在 middleware 目录执行
docker compose -f D:\middleware\docker-compose.yml -p middleware up -d
```

如果没有 docker-compose 文件，手动创建各容器（见下方）。

---

## 手动创建容器命令

```bash
# MySQL（端口 3307，避免和 Windows MySQL 冲突）
docker run -d --name mysql -p 3307:3306 \
  -v "D:\middleware\mysql-data:/var/lib/mysql" \
  -e MYSQL_ROOT_PASSWORD=root123 \
  --restart=unless-stopped mysql:8.0

# MongoDB
docker run -d --name mongo -p 27017:27017 \
  -v "D:\middleware\mongo-data:/data/db" \
  --restart=unless-stopped mongo:7

# Redis
docker run -d --name redis -p 6379:6379 \
  -v "D:\middleware\redis-data:/data" \
  --restart=unless-stopped redis:7.2-alpine

# RabbitMQ（含管理面板）
docker run -d --name rabbitmq -p 5672:5672 -p 15672:15672 \
  --restart=unless-stopped rabbitmq:3.13-management

# MinIO
docker run -d --name minio -p 9000:9000 -p 9001:9001 \
  -v "D:\middleware\minio-data:/data" \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin \
  --restart=unless-stopped minio/minio server /data --console-address ":9001"

# RedisInsight（Redis 可视化管理）
docker run -d --name redisinsight -p 5540:5540 \
  --restart=unless-stopped redis/redisinsight:latest

# Neo4j（知识图谱）
docker run -d --name neo4j -p 7474:7474 -p 7687:7687 \
  -v "D:\middleware\neo4j-data:/data" \
  -e NEO4J_AUTH=neo4j/neo4j123 \
  --restart=unless-stopped neo4j:5.22

# Chroma（RAG 向量数据库）
docker run -d --name chroma -p 8000:8000 \
  -v "D:\middleware\chroma-data:/data" \
  --restart=unless-stopped ghcr.io/chroma-core/chroma:0.4.20

# SearXNG（搜索增强）
docker run -d --name searxng -p 8080:8080 \
  -v "D:\searxng-docker:/config" \
  --restart=unless-stopped searxng/searxng
```

## 排查备忘

- **连不上数据库？** 先检查是不是连到 Windows MySQL（3306）了，应该连 3307
- **Navicat 连不上？** 确认 Host 是 `127.0.0.1`（IP 格式，不要带空格），Port 是 `3307`
- **端口冲突？** 用 `powershell Get-NetTCPConnection -LocalPort 3306` 看谁占了端口
- **Docker 连不上？** 用 `docker ps` 确认容器 Running，再用 `docker logs <name>` 看日志
