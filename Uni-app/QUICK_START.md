## 📦 启动步骤

### 步骤 1: 安装依赖

```bash
cd Uni-app
npm install
```

**预计时间**: 2-5 分钟

### 步骤 2: 配置后端地址

编辑 `utils/api.js` 文件，修改第 7 行：

```javascript
// utils/api.js
const BASE_URL = 'http://localhost:8888/api' // 改为实际的后端地址
```

**注意事项**:
- 开发环境：`http://localhost:8888/api`
- 测试环境：`http://your-test-server.com/api`
- 生产环境：`https://api.snake-sling.com/api`

### 步骤 3: 运行项目

#### 方式一：使用 HBuilderX（推荐）

1. 打开 HBuilderX
2. 文件 → 打开目录 → 选择 `Uni-app` 文件夹
3. 点击菜单「运行」→「运行到小程序模拟器」→「微信开发者工具」
4. 首次运行会自动打开微信开发者工具

#### 方式二：使用命令行

```bash
# 运行到微信小程序
npm run dev:mp-weixin

# 或运行到 H5（浏览器）
npm run dev:h5
```

### 步骤 4: 微信开发者工具配置

1. **导入项目**:
   - 打开微信开发者工具
   - 选择「导入项目」
   - 项目路径：`C:\Users\Asuka\Desktop\1\Sling\Uni-app`
   - AppID：填写你的小程序 AppID（测试可选「测试号」）

2. **配置不校验域名**（开发阶段）:
   - 点击右上角「详情」
   - 勾选「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」

3. **编译运行**:
   - 按 Ctrl+B 或点击「编译」按钮
   - 在模拟器中查看效果

## ⚠️ 常见问题

### 1. 图片上传失败

**原因**: 后端服务未启动或跨域配置问题

**解决方案**:
```bash
# 检查后端服务
cd ..
.\启动脚本.bat

# 确认网关地址配置正确
# utils/api.js 第 7 行
const BASE_URL = 'http://localhost:8888/api'
```

### 2. 定位功能不可用

**原因**: 未授权地理位置权限

**解决方案**:
- 微信开发者工具 → 详情 → 勾选「使用地理位置」
- 真机测试时需要在微信小程序后台申请地理位置权限

### 3. TabBar 图标不显示

**原因**: 图标文件缺失或路径错误

**解决方案**:
```bash
# 运行图标生成脚本
python generate_icons.py

# 或手动准备图标文件到 static/tabbar/ 目录
```

### 4. API 请求失败 401

**原因**: Token 过期或未登录

**解决方案**:
- 清除本地存储后重新登录
- 检查后端 token 验证逻辑
- 确认登录接口返回正确的 token

### 5. 样式不生效

**原因**: uni-app 样式限制或 scoped 属性问题

**解决方案**:
- 避免使用深度选择器
- 使用全局样式变量
- 检查 CSS 语法是否正确
