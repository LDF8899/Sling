处理逻辑补充
错误处理：
若StatusCode=429（频率超限），使用指数退避策略重试（最多 3 次，间隔 1s、2s、4s）；
若Animals数组为空（识别失败），返回预设提示 “无法识别，请上传更清晰的图片或补充文字描述”；
若最高置信度＜0.5，自动触发 “图片 + 文本” 混合识别流程（提示用户补充文字描述）。
匹配优化：结合Taxonomy和Features与snake_info表的characteristics进行模糊匹配（如通过LIKE '%绿色%环纹%'），提升跨名称别名的匹配准确率（如 “青竹蛇” 与 “竹叶青”）。

// 1. 图片识别流程（补充错误处理与重试）
public RecognitionResult recognizeByImage(String imageBase64, Long userId) {
    // 图片预处理：压缩至2MB以内（避免模型调用失败）
    String compressedImage = ImageUtils.compress(imageBase64, 2 * 1024 * 1024);
    if (compressedImage == null) {
        throw new BusinessException("图片格式错误，请上传JPG/PNG");
    }

    // 调用豆包视觉模型（带重试）
    VisionResponse visionResp = retryTemplate.execute(context -> {
        return doubaoVisionClient.recognize(compressedImage, "reptile");
    }, 3);  // 最多重试3次

    // 处理模型响应
    if (visionResp.getResponseMetadata().getStatusCode() != 200) {
        String errorMsg = visionResp.getResponseMetadata().getError().getMessage();
        log.error("视觉模型调用失败：{}", errorMsg);
        throw new BusinessException("识别服务暂不可用，请稍后再试");
    }

    // 匹配数据库蛇类（结合特征模糊查询）
    List<SnakeCandidate> candidates = visionResp.getResult().getAnimals().stream()
            .filter(animal -> animal.getConfidence() >= 0.5)  // 过滤低置信度结果
            .map(animal -> {
                SnakeInfo snake = snakeInfoMapper.matchByFeatures(
                    animal.getFeatures().getColor(),
                    animal.getFeatures().getPattern()
                );
                return new SnakeCandidate(
                    snake.getSnakeId(),
                    snake.getSnakeName(),
                    snake.getImageUrl(),
                    animal.getConfidence()
                );
            })
            .collect(Collectors.toList());

    // 记录识别记录（含模型元数据）
    RecognitionRecord record = new RecognitionRecord();
    record.setUserId(userId);
    record.setImagePath(uploadImage(compressedImage));
    record.setSnakeId(candidates.isEmpty() ? null : candidates.get(0).getSnakeId());
    record.setRecognitionResult(JSON.toJSONString(candidates));
    record.setRecognitionMetadata(JSON.toJSONString(
        Map.of("requestId", visionResp.getResponseMetadata().getRequestId(),
               "modelVersion", "2024-08-01")
    ));
    recognitionRecordMapper.insert(record);

    return new RecognitionResult(record.getId(), candidates);
}

// 2. 生成应急方案流程（补充医院距离计算）
public EmergencyPlan generatePlan(Long confirmedSnakeId, Double userLat, Double userLng) {
    // 1. 查询数据库应急信息
    SnakeEmergencyInfo emergencyInfo = snakeEmergencyMapper.selectBySnakeId(confirmedSnakeId);
    if (emergencyInfo == null) {
        throw new BusinessException("未找到该蛇类的应急信息");
    }

    // 2. 调用语言模型生成自然语言方案
    String systemPrompt = String.format("基于以下数据生成急救方案：%s", JSON.toJSONString(emergencyInfo));
    ChatResponse chatResp = doubaoChatClient.complete(systemPrompt, "生成应急方案");
    String firstAidSteps = parseSteps(chatResp.getResult().getChoices().get(0).getMessage().getContent());

    // 3. 查询附近有对应血清的医院（结合地图API计算距离）
    List<HospitalSerumDTO> hospitals = serumInventoryMapper.selectHospitalsWithSerum(confirmedSnakeId);
    List<NearbyHospital> nearbyHospitals = hospitals.stream()
            .map(hospital -> {
                // 调用高德地图API计算用户与医院的直线距离
                double distance = MapApi.calculateDistance(
                    userLat, userLng,
                    hospital.getLatitude(), hospital.getLongitude()
                );
                return new NearbyHospital(
                    hospital.getHospitalName(),
                    hospital.getAddress(),
                    String.format("%.1f公里", distance),
                    hospital.getContactInfo()
                );
            })
            .sorted(Comparator.comparing(NearbyHospital::getDistance))  // 按距离排序
            .limit(3)  // 返回最近3家
            .collect(Collectors.toList());

    return new EmergencyPlan(
        emergencyInfo.getSnakeName(),
        emergencyInfo.getVenomType(),
        emergencyInfo.getSymptomDescription(),
        firstAidSteps,
        emergencyInfo.getMedicalAttention(),
        nearbyHospitals
    );
}
六、缓存与日志设计（细化版）
1. 缓存策略补充
缓存键设计：
问答缓存：qa:{md5(question)}（MD5 哈希避免长键存储）
蛇类应急方案缓存：emergency:plan:{snakeId}
更新机制：
当snake_emergency_info表更新时（如通过后台管理接口），主动删除对应snakeId的方案缓存；
问答缓存设置 24 小时过期，同时提供手动清理接口（供管理员操作）。
缓存穿透防护：
对无效请求（如 “被龙咬伤怎么办”），缓存空结果（有效期 10 分钟），避免重复调用模型。
