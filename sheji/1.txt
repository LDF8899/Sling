三、 加入 Redis 的详细实施方案
Redis 的引入并非简单的缓存，而是作为系统的“高速反应堆”：

1. 核心急救数据缓存 （Hash 结构）
将 355 种蛇的核心实体结构化存入 Redis。

说明：sling:snake:info:{snake_id}

价值：{ "name": "银环蛇", "toxin": "神经毒", "level": "红色/重度", "first_aid": "..." }

优势： 响应时间由 MySQL 的 50-100ms 降至 1-5ms。

2. 医院与血清空间索引 （Geo 结构）
利用 Redis 的 存储全国 24 小时急诊且储备蛇毒血清的医院经纬度。GEOADD

指令： 搜索用户坐标附近 50km 内的血清点。GEORADIUS

意义： 在伤者中毒、神志模糊的黄金时间内，提供瞬间定位。

3. 数据同步与热更新 （Pub/Sub）
版本控制： 当你通过 Python 脚本爬取了新的物种图片或更新了资料，通过 Redis 广播给所有微服务节点，实现数据无感更新。

四、 优化优先级建议

第一步 （P0）：数据结构化。将资料中的 355 种蛇分类存入 Redis Hash，并同步生成小程序本地 JSON 。


第二步 （P1）：链路重构。修改 recognition-service，在 AI 返回蛇名后，第一时间从 Redis 取出急救卡片返回 。

第三步 （P2）：可视化增强。在前端集成 Cytoscape.js，调用后端封装好的图谱关系接口进行渲染。
